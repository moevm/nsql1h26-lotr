# Каталог персонажей вселенной Властелина Колец

## Прототип "Анализ" [v0.8]

### Запуск проекта

1. Клонируйте репозиторий и перейдите в директорию проекта:
```bash
git clone https://github.com/moevm/nsql1h26-lotr.git
cd nsql1h26-lotr
```

2. Создайте файл `.env` в корне проекта (можно скопировать из `.env.example`):
```bash
cp .env.example .env
```
Значения можно оставить по умолчанию.

3. Соберите и запустите контейнеры:
```bash
docker compose build --no-cache && docker compose up
```
Дождитесь готовности всех сервисов. Не бойтесь спама в логах, это шалит OGM neomodel, и однажды мы разберёмся как это отключить.

4. Откройте приложение в браузере: `http://localhost:8081`.

### Отладочные пользователи

При первом запуске создаются два пользователя.

| Логин | Пароль | Роль | Возможности |
| ----- | ------ | ---- | ----------- |
| `viewer` | `viewer123` | `viewer` | Чтение каталогов и страниц (доступно также анонимно), лайки |
| `admin` | `admin123` | `admin` | + Создание, редактирование, удаление сущностей, просмотр всего |

Регистрация тоже работает (как минимум, должна :) ). Пользователь создаётся с ролью `viewer`, пока что эту роль никак изменить нельзя

### Что было реализовано

#### Массовый импорт / экспорт
Админу доступна кнопка для иморта/экспорта всего содержимого в формате json.

#### Комментарии

Под статьями появились простенькие комментарии, `viewer` может написать комментарий, изменить или удалить свой комментарий. `admin` может удалять чужие комментарии.

#### Аналитика и статистика

Реализована кастомная статистика с фильтрацией, глобальная статистика по всей вики, возможность посмотреть граф связей для персонажа

#### Профили пользователей

Теперь можно перейти на профиль пользователя (например. нажав на его ник в комментарии). Там доступна информация о пользователе и список лайкнутых страниц. `admin` может посмотреть список всех пользователей (кнопка троеточие), изменить или удалить профиль другого пользователя.

### Не реализовано, но будет

### Известные ограничения

* Фильтры по связям всё ещё требуют точного слага, а фильтр по датам реализован через поиск по подстроке. Это осталось с прошлой итерации, мы знаем об этим проблема, и исправим. Просто пока не нашли времени.
* Реализован не весь функционал в разделе аналитики.
