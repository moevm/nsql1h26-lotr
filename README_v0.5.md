# Каталог персонажей вселенной Властелина Колец

## Прототип "Хранение и представление" [v0.5]

### Запуск проекта

1. Клонируйте репозиторий и перейдите в директорию проекта:
```bash
git clone https://github.com/moevm/nsql1h26-lotr.git
cd nsql1h26-lotr
```

2. Создайте файл `.env` в корне проекта (можно скопировать из `.env.example`):
```bash
cp .env.example .env
```
Значения можно оставить по умолчанию.

3. Соберите и запустите контейнеры:
```bash
docker compose build --no-cache && docker compose up
```
Дождитесь готовности всех сервисов. Не бойтесь спама в логах, это шалит OGM neomodel, и однажды мы разберёмся как это отключить.

4. Откройте приложение в браузере: `http://localhost:8081`.

### Отладочные пользователи

При первом запуске создаются два пользователя.

| Логин | Пароль | Роль | Возможности |
| ----- | ------ | ---- | ----------- |
| `viewer` | `viewer123` | `viewer` | Чтение каталогов и страниц (доступно также анонимно), лайки |
| `admin` | `admin123` | `admin` | + Создание, редактирование, удаление сущностей, просмотр всего |

Регистрацию делать было необязательно, но оказалось несложно, так что можете попробовать создать своего пользователя. Он создастся с ролью `viewer`.

### Что было реализовано

#### Аутентификация
* Вход: форма логина (доступна с главной страницы в навбаре).
* Профиль: кнопка в навбаре, доступно редактирование профиля, отображаются лайкнутые страницы.
* Выход: кнопка в профиле.

#### Каталоги сущностей

Если вы вышли из главного меню, то туда можно вернуться нажатием на лого в левом верхнем углу.

Навигационное меню содержит ссылки на доступные каталоги сущностей.

В каждом каталоге реализовано:
* Список сущностей с пагинацией
* Многокритериальная фильтрация (колонка справа). Применить фильтры можно нажатием кнопки внизу списка фильтров. Замечания: у JOIN-фильтров в скобках указано `(slug)` - они работают, но вводить нужно не имя сущности, а её слаг. Это временные меры, к следующей итерации это будет реализовано по-человечески. Фильтр по датам (а даты внутри вселенной свои, не от рождения Христова) реализован временно как поиск по подстроке - на следующей итерации заменим на более привычный "от-до".
* Создание сущности - кнопка `Add new ...`. Видна только `admin`, а у других прав на создание нет.

#### Страница сущности

При клике на любую карточку сущности в каталоге открывается её страница с полной информацией, счётчиком лайков (и возможностью лайкнуть, если вы залогинились). 

Для `admin` есть кнопки удаления и редатирования сущности. 

#### Лайки

Любой авторизованный пользователь может поставить/убрать лайк на странице сущности. Список лайков отображается в профиле пользователя.

#### Поиск

Поле поиска в навбаре, поддерживается поиск всех сущностей по имени (одному из).

### Известные ограничения

* Фильтры по связям треюут точного слага (например, `?race=hobbits`) - нет автодополнения в интерфейсе (будет в 0.8).
* Neo4j спамит логами в консоли - на работу это никак не влияет, просто не очень красиво :(.
